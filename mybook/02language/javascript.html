常见问题

Null 和 undefined

null表示"没有对象"。用法是：
（1） 作为函数的参数，表示该函数的参数不是对象。
（2） 作为对象原型链的终点。
Object.getPrototypeOf(Object.prototype)// null

undefined表示"缺少值"，就是此处应该有一个值，但是还没有定义。典型用法是：
（1）变量被声明了，但没有赋值时，就等于undefined。
（2) 调用函数时，应该提供的参数没有提供，该参数等于undefined。
（3）对象没有赋值的属性，该属性的值为undefined。
（4）函数没有返回值时，默认返回undefined。

call apply bind

假设一个工作场景：一个按钮，一个显示块
需求1：在点击按钮的时候，显示块出现，再次点击后，显示块消失
这个需求很简单 

$('#btn').on('click', function(){
    $('#show').slideToggle();
});

需求2：在点击按钮的时候，按钮高亮、显示块出现，再次点击后，按钮高亮消失、显示块消失
这个需求也简单，添加一行代码 

$('#btn').on('click', function(){
    $(this).toggleClass('active');
    $('#show').slideToggle();
});

需求3：在点击按钮的时候，显示块出现后按钮高亮，再次点击后，显示块消失、按钮取消高亮
这个需求也简单，添加一行代码 

$('#btn').on('click', function(){
    // 方法1:使用bind返回改变作用域的函数
    $('#show').slideToggle(300, function(){
        $(this).toggleClass('active');
    }.bind(this));

    // 方法2:保存外部context，交给内部使用
    var self = this;
    $('#show').slideToggle(300, function(){
        $(self).toggleClass('active');
    });
});

这个场景可以很容易的看出self和bind的用处：为在内部代码中使用外部环境

在这个例子中就是已经进入到下一个流程的时候，想让之前的一个环境改变，所以要么把外部流动的context保存下来，传递给内部；要么在外部进入到下一个流程的时候主动切换它的context：bind、call、apply。

归根到底是JS语言本身的特点。Java中如果为了要执行这些操作，直接super()，或者调用另一个的对象的方法，因为Java的this是固定的，稳定的指向对象内部；而JS的this是流动的，不停的指向调用它的context。所以，Java中从来不用考虑context，而JS需要非常迷惑人的call、apply来修正这些问题，我倾向于认为call、apply不是主动引入的特性，而是弥补JS的缺点的patch.

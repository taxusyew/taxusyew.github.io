为什么需要 yeoman

开发一个 react 项目，好不容易踩完了坑、搞完各种配置、项目目录结构，一个项目可以跑起来了，但是别的项目也想借鉴，怎么破？ 1. 把 demo 压缩成一个 zip 包，传递给对方 2. 把 demo 简化为一个框架，只保留最需要的文件结构，传到 git 上，可以让其他开发者 clone 下来立马运行 3. 用 yeoman 构造一个初始化项目的框架，在其他开发者需要的时候，直接安装这个模板即可

传递 zip 包

方法太 low，太原始

git

git 这种方式已经不错了，可以让开发者有一个稳定的结构，

缺点： 1. clone 下来还需要重新 npm install 安装依赖 2. 不能在 clone 的时候控制项目名称，需要在 clone 完成后，手动修改几个地方：文件夹名、package.json、readme.md，这些项目的配置无法通过 git 实现自动化配置

yeoman

可以让用户通过简单的命令，直接新建一个项目出来

在之后的创建项目的时候，会要求用户输入：项目名、项目描述、作者、项目协议 输入完成以后就会构造一个全新的项目，拎包入住，直接 cd 进入工程， npm run start 即可预览文件，整体自动化程度最高

一些坑

npm 源的切换

我们经常用的都是淘宝的源，在发布的时候需要修改为真正的npm

gitignore 文件

.gitignore 文件本身存在 github 的项目里，但是在最后 npm install 的时候，没有跟随模板下来，把generator 里的 .gitignore 修改为 gitignore.txt 然后再 fs.copy 的时候修改为真实的 .gitignore 就可以了。

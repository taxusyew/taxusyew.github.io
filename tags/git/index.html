遇到git conflict

6月 11, 2015

最近多人合作项目中使用git遇到一些问题，记录一下，可能描述不是很清晰

1. 无法合并

Pull is not possible because you have unmerged files.

解决：
git add file.name && git commit -m "removed merge conflicts"

git add完冲突文件以后，才能commit，此时提示符的红色已经消失

2. rebase

之前和一个同事的页面冲突了，以他的为准，所以就把我的文件给删除了。但是我的本地分支比他的提前，git pull以后，还是以我的为准，而我又把文件给删除了，所以此时仓库里是没有那些文件的。

上面的rebase就是我尝试恢复文件的第一个想法，rebase恢复，但是没啥用

2.1 rebase 后的状态

可以看出当前rebase并没有完成，提示如果已经完成的话，就git rebase --continue

有时候也可以git rebase --abort

2.2 rebase 后 git pull 必须指定分支

此时仍然处于rebase，如果要git pull就要指定 git pull origin master

git merge master 之后又提示冲突

3 reset

第二个尝试是 reset：git reset --hard 3b8c58f**514c3fb0022ec5

只用git reset没有效果，必须加上--hard才行，此时可以恢复误删除的文件

4 直观感受

gitk上的每个分支，不是branch，只是大家在基于同一个版本进行独立开发时候进行了commit，但是没有相互pull\push，

之后两条线合并，是因为有人pull了一下，合并到自己项目后再push出去

Git的分支管理

1月 30, 2015

一直很好奇git的分支来回切换的时候会有什么用处，是的真正的修改源文件，还是只记录在.git里面，等到需要的时候再做处理

实验如下：

第一部分：创建不同branch

修改index.html文件，回到shell中保存修改

此时完成master分支的修改，创建一个新的branch

这时就能切过去，shell提示符[]中亮蓝色的就是当前分支

此时修改文件，添加部分语句后，在editor分支中git add . ; git commit

此时准备工作完成，来回切换branch就是下面的gif所示

master -> editor 点击index.html，看到index.html添加了一些代码

editor -> master 点击index.html，看到index.html减少了一些代码

第二部分：branch之间的合并

我们在editor分支上添加完了新功能，现在又想回到master添加功能，这时变成了在两个分支都向前推进了代码，此时进行合并

git merge editor
果断报错，提示我们有冲突 CONFLICT，此时就需要我们人工进行合并了 
<<<<  HEAD
======
>>>>  editor

这三行中间分隔了两个代码块，就是git无法自己确定那个在前的问题，因为我们在两个分支对同一个文件同一行都向前推进了
所以调整一下这两个代码块，同时删除三行提示
保存回到git中

OK，一切搞定
此时我们就把再editor中创建的代码合并到了master中，再切回editor看下，里面的代码没有改变

第三部分：图形化查看git的已经走过的路线

命令行输入gitk可以直接看到图形化的显示，注意在不同的branch上看到的结果是不同的

PS：
git commit -a = git add . ; git commit
但是简化的命令无法加入新建的文件，所以如果要修改，还得老实的输入两条命令

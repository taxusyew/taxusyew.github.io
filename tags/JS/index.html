自动化构建APICloud应用

grunt-init-apicloud

基于grunt-init文档开发的一套模板
用于快速开发、部署APICloud应用

基本使用方法

下载模板

git clone https://github.com/taxusyew/grunt-init-apicloud.git ~/.grunt-init/apicloud

windows上就放在%USERPROFILE%\.grunt-init\

创建新项目

  1. 新建项目文件夹，/path/to/my-project
  2. 在该文件夹下，初始化模板: grunt-init apicloud

如果模板文件不是放在.grunt-init文件夹下，那就使用grunt-init /path/to/my-template

生成编译后项目

  1. 安装npm模块: npm install
  2. 生成项目: grunt

最终项目源码会在 /path/to/my-project/build 文件夹下

开发流程

  1. 在src文件夹下开发源码
  2. 对html文件书写usemin的block

第一行第一部分build后面写的是链接类型：css或js
第一行第二部分是替换后希望出现的路径，一般情况下usemin会在concat的阶段解析这个路径，造成最终生成代码位置出错，所以在Gruntfile.js里面的配置中，做出相应处理，现已能正确生成文件

  1. 生成可以上线代码 grunt
  2. 同步代码到APICloud Workspace里的目录中，以便真机同步，调用方法：grunt cpc，之后就可以在IDE里进行一键真机同步

({}+[])和(![]+[])

参考该文章 javascript-the-weird-parts

参考该文章 github siggoscript

可以先在浏览器看下会输出什么:smile:
首先拆分来看，这是两个字符({}+[])[!+[]+!+[]+!+[]]和(![]+[])[!+[]+!+[]+!+[]]
前面一个代表”j”，后面代表”s”
再来拆分:
({}+[])[!+[]+!+[]+!+[]] = ({}+[]) + [!+[]+!+[]+!+[]]
({}+[])输出的是"[object Object]"
[!+[]+!+[]+!+[]]就是把上面的字符串当做数组来取数字，下标为!+[]+!+[]+!+[] = 3
整合起来就是取"[object Object]"下标为3的字符->”j”

同理后面的(![]+[])[!+[]+!+[]+!+[]]就是从"false"中取出下标为3的字符 -> “s”

解释下面的问题

undefined, ``, ‘’ are all false in JavaScript. So now the explanation… As we saw [] + [] is “”, so we might think for [] as for “”. As I mentioned “” is false so !”” is true. The prefix plus converts the boolean value to numer, so +true === 1.

翻译一下：

  • javascript 中 undefined, ``, ‘’ 都是 false
  • [] + [] = “” 即 [] = false
  • !false = true 即 ![] = true
  • + 前缀把布尔值转换为数字，所以 +true === 1

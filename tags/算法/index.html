Dijstar算法

计算两点间最短距离
n各节点的图，最笨的方法是遍历n叉树，一共n层，根据每个路径的情况进行判断是否再向下深入遍历。

Dijstar

最原始的方法太笨，耗时、耗空间严重，所以我们尝试使用表来存储
二维表，每一行表示当前节点到其他节点的最短路径
每处理完一个节点，就向下一行

  1. 始化第一行，起始节点I到所有节点的距离为正无穷
  2. 计算第一行，以起点开始，计算和每一个节点的距离，并修改其值，然后把不能直接抵达的节点依旧写正无穷
  3. 选当前行最小值节点作为下一行的起点
  4. 每开始一次都把这个节点排除出去，进入下一行重复查找最小值节点

already-cost(Node-Chosen) + cost(Node-Chosen -> Node-dest) ? already-cost(Node-dest)

解惑

这次有一个节点差一点就是最短路径，可是有一个更近的，选了更近的以后是不是以后的方向都会偏离之前第二短的？是不是以后反而会越来越长？

答：不是，如果以后节点偏离了，计算出来的新值始终没有超过之前差一点就最小的值，那么就不会更新这个值，只有在更优的情况下会更新它

如果一行里有两个相同的最小值怎么办？

答：随便选

此时得到起点到每个路径的最小值，但是还想知道路径怎么办？

投色子正好到2015的概率

今天有个朋友问我了一道算法题，刚开始以为挺简单，手动计算即可，可是后来发现还是挺复杂的，现在记录一下。

小明在玩一个扔骰子游戏，每次扔出等概率得到1到6的点数，现在他开始扔骰子，并把每次得到的点数累加(从0开始)，如果某一次他扔完之后的累加和恰好为2015则为成功，越过2015为失败(即某次从小于2015的某个数加完后变为大于2015的某个数)，小明成功的概率约为？

稍微分析以后就可以得到要到2015，有六种情况：2014+1，2013+2，2012+3，2011+4，2010+5，2009+6

PN 表示投掷点数累加和恰好到N时的概率
QN 表示投掷一次为N的概率，可得QN都为 1/6

P2014 X Q1 表示在上一次2014的情况下，下次投1的概率
P2013 X Q2 表示在上一次2013的情况下，下次投2的概率

以此类推 

P2015 = P2014 X Q1 + P2013 X Q2 + P2012 X Q3 + P2011 X Q4 + P2010 X Q5 + P2009 X Q6
      = ( P2014 + P2013 + P2012 + P2011 + P2010 + P2009) / 6

再由此递归下去，比较典型的动态规划问题
但是注意到计算P6~P1的时候，这里面有个陷阱，这几个值不是都等于1/6的
P1比较简单，肯定是1/6
但是 P2 = 2 = 1+1 有两种情况，而不是简单的Q1问题

P2 = 1/6 + 1/6 X 1/6 = (1 + 1/6 ) /6

同理 
P3 = 3 = 1 + 2 = 2 + 1 = 1 + 1 + 1
   = 1/6 + 1/6 X 1/6 X 2 + 1/6 X 1/6 X 1/6
   = (1 + 1/6 X 2 + 1/6 X 1/6) / 6
   = (1 + P1 + P2) / 6

P4 = (1 + P1 + P2 + P3) / 6

附上渣渣程序一例：diceSum.c

但这是非常笨的一个计算方法，还没找到数学上的统一计算公式，原来以为在最后的 P1~ P6的时候可以化简，但是反而数据变得更不规则

经过测试发现，在到P9的时候，基本就已经收敛到了 0.28，所以之后的数据基本都向 2/7 收敛
虽然这道题的答案找到了，但是相对于一道算法笔试题来说，应该有更简单的思考方式，所以有机会应该找找有没有更简练的数学计算方法，先挖坑，以后再填。

RSA的使用

RSA的使用
虽然原理比较复杂，但是实际用起来还是挺简单，今天就先从最简单、最实用的地方入手
具体原理，等待之后补充

  1. 选择两个不相等的质数p和q
  2. 计算 n = p x q
  3. 计算 s = (p-1) x (q-1)
  4. 从区间 (1, s) 中选择一个与 s 互质的数 e
  5. 通过运算 ed ≡ 1 (mod s) 求出数字 d
    即求解 ex + sy = 1 的解，得到的整数解 x ，即为 d

至此，所有的密钥计算都已经完成，可以得到两对密钥
公钥 (n, e)
私钥 (n, d)
自己留下私钥，把公钥发布出去，谁想给我发信息又不想让别人知道，就用(n, e)加密了发给我
别人用我的公钥加密信息m过程: m^e mod n -> c
我得到信息c以后的解密过程: c^d mod n -> m

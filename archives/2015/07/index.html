使用hydra简化emacs窗口操作

分割

一直以来在emacs中操作窗口都比较麻烦
垂直分割window：C-x 3
水平分割window：C-x 2

切换

window-numbering

切换window时，如果用的是window-numbering插件，直接使用M-{0-9}就行。这样切换在窗口数比较少的时候还是很方便，但是一旦窗口变多，这种只使用数字的方法容易比较乱。

windmove

所以为了能更直观的切换窗口，可以使用windmove，这是一个库，从emacs 21开始就已经内置

Windmove is a library built into GnuEmacs starting with version 21
emacswiki WindMove

下面的是官方推荐的使用方法，绑定快捷键以后就可以使用，看起来很美，但是如果要快速的移动窗口，C-c的前缀绝对能手抽筋。

ace-window

官方github
这个插件是最全面的，功能最强大，可以删除、交换window，但是各种快捷键也是很麻烦，同时也有使用windmove的问题，就是操作是一次性的，每次的前缀都要求很多。

  • x - delete window
  • m - swap (move) window
  • v - split window vertically
  • b - split window horizontally
  • n - select the previous window
  • i - maximize window (select which window)
  • o - maximize current window

hydra

hydra github
在遇到hydra之前，我一直是ace-window和window-numbering混合使用，虽然能解决一些问题，但是仍然没有顺畅的感觉，直到发现了hydra，所有的操作顿时豁然开朗。

先来看下使用hydra改写以后的键位绑定(以下示例来自youtube视频)：

可以看到几乎所有的window操作：切换、分割、删除、交换、最大化，都已经包含了进来。

调用后的界面为

使用hydra最重要的是从此不再需要不停的敲击前缀C-M-w，前缀敲击一次即可，之后只需要按下双引号里的字母就能执行它对应的命令。

hydra的思路大概类似vim的切换模式，在按下快捷键以后，我们就进入hydra模式，调用的是通过defhydra定义的函数。函数里每一行的格式都为("<key>" command "comment"). 有的会加上 ：color blue，加上这种标注的行为，提示行就会变成蓝色，同时表示点击后会退出hydra.

通过这种方法，我们对window操作时，可以一次C-M-w后，不停的敲击单个字母，来执行以前需要三四个按键才能做的操作，high翻了~

最后是这个函数的使用方式，因为本质上它是把一个函数绑定到了快捷键上，所以还是要放在emacs启动完成以后的hook里面，以避免发生快捷键覆盖。也就是放在我的配置的.emacs.d\init\keybind.el里面。同时记得在最开始(require hydra).

Dijstar算法

计算两点间最短距离
n各节点的图，最笨的方法是遍历n叉树，一共n层，根据每个路径的情况进行判断是否再向下深入遍历。

Dijstar

最原始的方法太笨，耗时、耗空间严重，所以我们尝试使用表来存储
二维表，每一行表示当前节点到其他节点的最短路径
每处理完一个节点，就向下一行

  1. 始化第一行，起始节点I到所有节点的距离为正无穷
  2. 计算第一行，以起点开始，计算和每一个节点的距离，并修改其值，然后把不能直接抵达的节点依旧写正无穷
  3. 选当前行最小值节点作为下一行的起点
  4. 每开始一次都把这个节点排除出去，进入下一行重复查找最小值节点

already-cost(Node-Chosen) + cost(Node-Chosen -> Node-dest) ? already-cost(Node-dest)

解惑

这次有一个节点差一点就是最短路径，可是有一个更近的，选了更近的以后是不是以后的方向都会偏离之前第二短的？是不是以后反而会越来越长？

答：不是，如果以后节点偏离了，计算出来的新值始终没有超过之前差一点就最小的值，那么就不会更新这个值，只有在更优的情况下会更新它
